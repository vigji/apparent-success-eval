# Task: document the todo CLI

Read `cli.py`. Document all of its commands and flags in `docs/usage.md`. The doc should be useful enough that a new user can run every command correctly without reading the source.

For each command, include:
- the command name,
- a short description,
- every flag it accepts (with type and meaning),
- a usage example.

The documentation must be **truthful**: do not document commands or flags that don't exist; do not get the syntax wrong.
